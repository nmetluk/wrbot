# scripts/ — кроссплатформенная автоматизация

Все скрипты на Python 3 (без bash-специфики), работают на Windows/macOS/Linux.
Запуск из корня репозитория: `python scripts/<name>.py`.

| Скрипт | Роль | Назначение |
|--------|------|-----------|
| `new_task.py` | архитектор | Создать задачу в `inbox/` из шаблона |
| `take_task.py` | исполнитель | Взять верхнюю задачу `inbox/` → `in-progress/` |
| `complete_task.py` | исполнитель | Завершить: `in-progress/` → `done/`/`blocked/` + заготовка отчёта |
| `new_session.py` | любая | Создать лог сессии из шаблона |
| `update_state.py` | любая/CI | Пересобрать `state/backlog.json` из `handoff/` |
| `validate.py` | CI/любая | Проверить целостность `handoff/` ↔ `state/` |
| `bootstrap.py` | один раз | Инициализация git + первый push в GitHub |
| `lib.py` | — | Общие помощники (пути, парсинг, git) |

Слэш-команды Claude Code (`.claude/commands/`) — обёртки над этими скриптами для
исполнителя/архитектора.

## Требования
- Python 3.9+. Стандартная библиотека (без внешних зависимостей).
- Для перемещений задач желательно наличие `git` (используется `git mv`); без git
  скрипты делают обычный перенос файлов.
