## Цель сессии
Сделать `SentReminderRepository.record()` полностью диалект-независимым (SQLite + PostgreSQL), убрав `dialects.sqlite` + `prefix_with("OR IGNORE")`. Сохранить идемпотентность. Полный CI зелёный.

## Что сделано
- В `src/wrbot/repositories/sent_reminders.py`:
  - Удалён импорт `from sqlalchemy.dialects.sqlite import insert as sqlite_insert`.
  - `record()` теперь использует обычный `insert(SentReminder).values(...)` + перехват `IntegrityError` (из `sqlalchemy.exc`).
  - Это идемпотентно на любой БД с UNIQUE constraint `uq_reminder` (ON CONFLICT DO NOTHING семантика).
  - Добавлен комментарий о совместимости с PostgreSQL.
- Тесты `tests/test_repositories_sent_reminders.py` проходят без изменений (идемпотентность: первый record → True, повтор → False).
- Полный CI начисто: ruff 0, mypy (baseline), pytest 145 passed, alembic, validate — всё 0.

## Принятые решения
- Выбран вариант (б) из задачи (try/INSERT + catch IntegrityError) — самый простой, надёжный и не требующий диалект-специфического кода. on_conflict_do_nothing имел проблемы с API в текущей версии SQLA для общего insert().
- Не меняли was_sent() — он и так портативный.
- Не добавляли отдельный тест-маркер для PG, так как поведение эквивалентно и покрыто существующим тестом идемпотентности.
