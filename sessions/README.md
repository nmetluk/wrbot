# sessions/ — журнал сессий (append-only)

Каждая рабочая сессия (архитектора или исполнителя) оставляет здесь **один**
файл-лог. Это основной механизм передачи контекста между сессиями для человека
и для ИИ: прочитав последний лог, агент понимает, на чём остановились.

## Именование
`SESSION-YYYY-MM-DD-NN.md`, где `NN` — порядковый номер сессии за день (01, 02, …).
Пример: `SESSION-2026-05-31-01.md`.

## Правила
- Файлы **не редактируются задним числом** и не удаляются (append-only журнал).
- Лог создаётся из шаблона `docs/templates/session-log.template.md` (или командой
  `/end-session`, которая использует `scripts/new_session.py`).
- Лог фиксирует: роль, цель сессии, что сделано, какие задачи затронуты, какие
  решения приняты, что осталось/следующий шаг, ссылки на коммиты.
- Лог — это «человеческий» нарратив; машиночитаемое состояние живёт в `state/`.
