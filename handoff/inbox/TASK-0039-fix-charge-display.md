# TASK-0039: Реальные данные в сводке создания и карточке списания

## Цель (отзыв пользователей, баг)
Сейчас при подтверждении создания списания и при просмотре карточки выводятся **заглушки и сырые
значения** вместо реальных данных. Исправить на корректные имена/формат.

## Подтверждённые дефекты (по коду)
- `handlers/charges_create.py:_build_summary_text` — `wallet="выбран"` (хардкод!),
  `category="выбрана"/"пропущена"` (не имя!), `next_date` = ISO-строка (`2026-06-23`), хотя надо **ДД.ММ.ГГГГ**.
- `handlers/charges_list.py:show_charge_card` — `wallet=f"ID {charge.wallet_id}"`,
  `category=f"ID {charge.category_id}"`, `next_date=charge.next_date` (ISO), `period=charge.period`
  (сырое `monthly`), `notify="настроены"` (хардкод). Все TODO-заглушки.
- `new_charge_created` и `charge_paid_periodic` выводят `next_date` как ISO (объект date → str).
- `my_charges_item` использует `{wallet}`/`{next_date}` — тоже должны быть имя/ДД.ММ.ГГГГ.

## Решение
Создать **общий форматтер** `services/formatters.py` (или дополнить `services/dates.py`) с функциями:
- `format_date_ru(d: date) -> str` → `ДД.ММ.ГГГГ`.
- `format_period_ru(period: str) -> str` → одноразово/ежемесячно/ежеквартально/ежегодно
  (вынести единый маппинг; сейчас дублируется `_format_period` локально).
- `format_notify(...)` → человекочитаемо (глобальные / свои: дни / отключены).
- Хелпер карточки/сводки, который **резолвит имена кошелька и категории через репозитории**
  (WalletRepository/CategoryRepository), а не печатает id/заглушку.
Применить в: `charges_create.py` (сводка, `new_charge_created`), `charges_list.py`
(`show_charge_card`, `list_charges`/`my_charges_item`), `charge_paid_periodic`.

## Критерии приёмки (проверяемые)
- [ ] Сводка перед созданием: реальные **имя кошелька** и **имя категории** (или «—» если пропущена),
      дата в **ДД.ММ.ГГГГ**, период по-русски.
- [ ] Карточка списания: имя кошелька, имя категории (или «—»), дата ДД.ММ.ГГГГ, период по-русски,
      корректный текст уведомлений (не «настроены»).
- [ ] `new_charge_created` и `charge_paid_periodic`: дата в ДД.ММ.ГГГГ.
- [ ] Список «Мои списания»: дата ДД.ММ.ГГГГ, имя кошелька.
- [ ] Все тексты — через локали (`texts.py`), без хардкода строк в хэндлерах.
- [ ] Тесты форматтера **через реальный текст шаблона** (урок анти-дрейфа: хотя бы один рендер
      без мока `t()`/Texts); e2e через `dp.feed_update`: создать списание → в сводке видны
      реальные кошелёк/категория/дата ДД.ММ.ГГГГ.
- [ ] Весь CI начисто зелёный.
