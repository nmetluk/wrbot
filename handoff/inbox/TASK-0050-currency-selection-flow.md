# TASK-0050: Выбор валюты в потоке создания/редактирования (ADR-0013)

## Цель (срочный отзыв, часть 2/2)
Дать пользователю выбрать валюту списания: быстрые пресеты + механизм выбора любой из справочника.

## Решения владельца (зафиксированы)
- Механизм «другой» валюты — **список + поиск** (постраничный список всех + ввод кода/названия для фильтра).
- Валюта по умолчанию — **запоминать последнюю** использованную (`User.last_currency`, старт RUB).

## Реализация
- **Шаг валюты** в `NewChargeStates`: после `amount`, перед `wallet` (название→сумма→**валюта**→
  кошелёк→категория→период→дата→уведомления). Добавить состояние `currency` (+ под-состояние поиска).
- **Клавиатура валюты:** 5 пресетов RUB/EUR/USD/KZT/KGS (из `currencies.get_presets()`), преселект —
  `user.last_currency`; кнопка «🌐 Другая валюта».
- **«Другая» → список + поиск:**
  - постраничный инлайн-список всех валют (пагинация ◀️/▶️, callback с номером страницы),
    по кнопке на валюту — `{code} — {name}`;
  - подсказка «введите код или название для поиска» → текстовый ввод → `currencies.search(query)`
    показывает совпадения кнопками; пустой результат — понятное сообщение (через локаль).
  - выбор валюты (из пресета/списка/поиска) → сохранить в FSM, перейти к шагу кошелька.
- **Запоминание:** при создании списания обновлять `User.last_currency` выбранной валютой; при
  старте нового создания подставлять её как преселект.
- **Редактирование (TASK-0045 flow):** на шаге валюты показывать текущую валюту списания; дать сменить.
  Живая карточка (build_edit_live_card) — показывать валюту.
- **Подтверждение/создание:** `confirm_create_charge` сохраняет `currency`; сводка и карточка
  показывают валюту (через форматтер TASK-0049).
- Тексты — через локали; callback-префиксы НЕ должны конфликтовать (урок TASK-0008/0046:
  специфичные раньше широких; покрыть router-level тестом).

## Критерии приёмки (проверяемые)
- [ ] При создании списания есть шаг валюты: 5 пресетов + «Другая»; преселект = последняя валюта юзера.
- [ ] «Другая» → постраничный список всех валют (пагинация работает) И поиск по вводу кода/названия
      (напр. «GBP» или «pound») → выбор из совпадений. Невалидный/пустой ввод — понятное сообщение.
- [ ] Выбранная валюта сохраняется в списание; сводка/карточка/список показывают её; `User.last_currency`
      обновляется и подставляется в следующий раз.
- [ ] Редактирование позволяет сменить валюту; живая карточка показывает валюту.
- [ ] Callback-роутинг валютных кнопок без шадовинга (специфичные раньше широких); **router-level
      тест через `dp.feed_update`** (урок TASK-0046): пресет, открытие списка, пагинация, поиск→выбор.
- [ ] e2e создания со сменой валюты: создать списание в USD → в карточке «… USD»; повторное создание
      преселектит USD.
- [ ] Весь CI начисто зелёный.

## Ожидаемые артефакты
