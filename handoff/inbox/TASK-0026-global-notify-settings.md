# TASK-0026: Глобальные уведомления — редактирование времени и дней (FR-10)

## Цель
Реализовать пропущенный FR-10: в «⚙️ Настройки → 🔔 Глобальные уведомления» дать
изменять время уведомлений (`notify_time`) и глобальные дни напоминаний (`global_days`).

## Контекст / приоритет
- **Severity: RELEASE BLOCKER.** Найдено в финальном аудите (AUDIT-M5-2026-06-02): кнопка
  `settings_global` — заглушка `global_settings_stub`, `SettingsStates` закомментирован.
- ТЗ §3.3 (Глобальные уведомления: «Время уведомлений (по умолчанию 10:00)»;
  «Дни напоминаний (за 5, 3, 1 день — любые комбинации)»). FR-10.
- Готово: поле `users.notify_time`/`global_days` (схема), свип уже использует их (M4);
  `UserRepository`; паттерн ввода/FSM как в кошельках/tz (M2/0025).
- Уроки: специфичные callback'и + router-тест (TASK-0008); тексты в `texts.py`; моки autospec.

## Критерии приёмки (проверяемые)
- [ ] «🔔 Глобальные уведомления» открывает экран с текущими значениями: время (HH:MM) и дни,
      и кнопками их изменить (убрать заглушку `global_settings_stub`).
- [ ] **Изменение времени:** ввод `notify_time` в формате `ЧЧ:ММ` с валидацией (0–23:0–59),
      сохранение в `users.notify_time` (метод `UserRepository`), изоляция по tg_id.
- [ ] **Изменение дней:** выбор/ввод набора дней (минимум поддержать комбинации 5/3/1, лучше —
      произвольные положительные целые), сохранение в `users.global_days` (JSON), валидация
      (непустой список положительных чисел или явное «выключено»).
- [ ] `SettingsStates` (раскомментировать/добавить): `notify_time`, `global_days`. `/cancel` сбрасывает.
- [ ] Специфичные callback'и (напр. `gnotify_time`, `gnotify_days`, `gday_toggle_<n>`) — без
      перехвата чужих; **router-тест**; тексты только в `texts.py` (+ тест рендера).
- [ ] Изменения реально влияют на свип (время/дни) — покрыть тестом на уровне логики (due/выбор по времени).
- [ ] Весь CI начисто зелёный: `uv run` ruff format/check, mypy, pytest, alembic, validate.
