# TASK-0047: Получение ID группы (/getgrid + @упоминание)

## Цель (запрос пользователя)
Когда пользователь добавил бота в группу/канал-обсуждение, он должен в этой группе вызвать
**`/getgrid`** или **@упомянуть бота** и получить в ответ **ID чата**, чтобы вписать его в
настройки категории (TASK-0043, поле `notify_chat_ids`).

## Решение владельца (зафиксировано)
- Вызывать может **любой участник** группы — проверка прав/админ-статуса НЕ нужна.

## Контекст (по коду)
- Бот на long polling; фильтра «только приватные чаты» нет (`__main__.py`).
- Команды сейчас регистрируются только для `BotCommandScopeAllPrivateChats` (`__main__.py:67`).
- Privacy mode по умолчанию: в группах бот и так получает **команды** (`/cmd`, `/cmd@bot`) и
  **@упоминания** — отключать privacy НЕ требуется.

## Реализация
- Новый роутер `handlers/group.py` (включить в `__main__.py`), хэндлеры **скоупить на группы**:
  `F.chat.type.in_({"group", "supergroup"})` (+ при желании channel-обсуждения).
- **`/getgrid`**: `@router.message(Command("getgrid"), F.chat.type.in_({"group","supergroup"}))`
  → ответ с `message.chat.id` (моноширинным/копируемым), краткая подсказка «вставьте этот ID в
  Настройки → категория → Каналы для напоминаний». aiogram `Command` сам разбирает `/getgrid@bot`.
- **@упоминание**: хэндлер на групповые сообщения, где упомянут бот — проверять `message.entities`
  (`type == "mention"` с текстом `@<bot_username>`; и/или `text_mention` на `bot.id`). Username бота
  получить через `await bot.me()` (кэшировать). На совпадение — тот же ответ с ID. Если бота не
  упомянули — игнор (не отвечать на каждое сообщение; privacy mode и так фильтрует, но проверку
  упоминания оставить как защиту).
- Зарегистрировать `/getgrid` в меню команд групп: `set_my_commands([... getgrid ...],
  scope=BotCommandScopeAllGroupChats())` в `__main__.py` (приватный скоуп не трогать).
- Тексты — через локали (`texts.py`), RU.
- Учесть порядок роутеров: групповой роутер не должен конфликтовать с приватными FSM/командами
  (скоуп по chat.type решает; убедиться, что `/start` и пр. в группе не ломают логику — вне scope,
  но не регрессить).

## Критерии приёмки (проверяемые)
- [ ] В группе `/getgrid` (и `/getgrid@bot`) → бот отвечает ID этой группы (копируемый), с подсказкой.
- [ ] В группе @упоминание бота → тот же ответ с ID; на обычные сообщения без упоминания бот молчит.
- [ ] Команда `/getgrid` видна в меню команд в группах (group scope), приватное меню не изменилось.
- [ ] Любой участник может вызвать (без проверки прав).
- [ ] В приватном чате `/getgrid` ведёт себя осмысленно (либо отдаёт chat.id, либо подсказывает,
      что команда для групп) — на усмотрение, но без ошибки/зависания; `callback`/`answer` не нужен (это message).
- [ ] **Router-level тесты через `dp.feed_update`** (урок TASK-0046, НЕ прямой вызов хэндлера):
      апдейт с `chat.type="supergroup"`, отрицательный `chat.id`, текст `/getgrid` → ответ содержит id;
      апдейт с mention-entity на username бота → ответ; сообщение без упоминания → нет ответа.
- [ ] Весь CI начисто зелёный.
