# TASK-0005: Бот-UI справочников (Настройки → Кошельки/Категории)

## Цель
Реализовать пользовательский CRUD кошельков и категорий через inline-интерфейс в
разделе «⚙️ Настройки», поверх репозиториев и DI-сессии из TASK-0004 (FR-8, FR-9).

## Контекст
- ТЗ: §3.3 (Настройки → Кошельки/карты CRUD, Категории CRUD), §3.1 (главное меню).
- Требования: FR-8, FR-9, FR-13 (изоляция), FR-14 (русский), NFR-3.
- ADR: ADR-0003 (репозитории), ADR-0007 (long polling).
- Текущее состояние: `bot/handlers/start.py` (только /start,/help,/cancel — заглушки),
  кнопка `settings` (callback_data="settings") **без обработчика**; `bot/keyboards.py`,
  `bot/states.py` (пустые StatesGroup), `bot/texts.py` (класс `Texts`).
- Правила `CLAUDE.md`: хэндлеры тонкие, логика в сервисах/репозиториях; тексты — только
  через `texts.py` (не хардкод); FSM в `states.py`; клавиатуры в `keyboards.py`;
  anti-drift: моки со `spec`/autospec; **хотя бы один тест текста с реальным рендером**
  (если в строках есть подстановки `{name}` и т.п.).

## Критерии приёмки (проверяемые)
- [ ] **Меню «Настройки»:** обработчик callback `settings` открывает подменю с разделами
      «Кошельки/карты», «Категории» (и «Глобальные уведомления» — кнопка-заглушка для M3),
      кнопка «Назад» в главное меню.
- [ ] **Кошельки (FR-8):** список кошельков пользователя; «➕ Добавить» (FSM: запрос имени →
      создание через `WalletRepository`); по каждому — «✏️ Переименовать» (FSM) и
      «🗑 Удалить» (с подтверждением). Пустой список — понятный текст-подсказка.
- [ ] **Категории (FR-9):** тот же набор действий через `CategoryRepository`.
- [ ] **FSM-состояния** в `states.py` (напр. `WalletStates`, `CategoryStates` для add/rename);
      `/cancel` и кнопка «Отмена» сбрасывают состояние (реализовать TODO из `start.py`).
- [ ] **Изоляция tg_id (FR-13):** все операции — только над объектами текущего пользователя
      (через репозитории из TASK-0004); upsert пользователя (`get_or_create`) при первом обращении.
- [ ] **Лимиты/валидация** из TASK-0004: при превышении лимита или пустом/дублирующемся имени —
      понятное сообщение пользователю (не трейсбек).
- [ ] **Тексты — только в `texts.py`** (русские), включая шаблоны с подстановками
      («Кошелёк «{name}» добавлен» и т.п.).
- [ ] **Тесты:** хэндлеры (моки репозиториев со `spec`/autospec; проверка вызовов и веток —
      добавление/переименование/удаление/отмена, пустой список); **минимум один тест
      реального рендера текста** с подстановкой. Все callback/entry-пути покрыты.
- [ ] CI зелёный начисто (без `BOT_TOKEN`): ruff, ruff format, mypy, pytest, `alembic upgrade head`, `validate.py`.
- [ ] Документация: при необходимости обновить `docs/architecture/components.md` и `docs/commands.md`/README по новым экранам.

## Ожидаемые артефакты
- Код: `src/wrbot/bot/handlers/{settings,wallets,categories}.py`, дополнения в
  `keyboards.py`, `states.py`, `texts.py`; регистрация роутеров в `__main__`.
- Тесты: `tests/test_handlers_*.py`, тест рендера текста.

## Ограничения / заметки
- Не реализуй списания/уведомления/глобальные настройки уведомлений — это M3/M4
  (раздел «Глобальные уведомления» — заглушка).
- Тонкие хэндлеры: вся работа с БД — через репозитории из TASK-0004, не напрямую.
- Кроссплатформенно, секреты в `.env`. Перед `done`: отчёт, лог сессии (уникальный ID), state, push.
- Реальную проверку в Telegram UX желательно сделать руками (UX-баги не всегда ловятся unit-тестами).
