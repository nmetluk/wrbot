# TASK-0042: Эмодзи-иконки кошельков (ADR-0011)

## Цель (отзыв пользователей, п.5)
Дать визуально различать карты/кошельки: при добавлении и редактировании кошелька — выбор иконки
из готового пресета эмодзи; иконка показывается рядом с названием везде.

## Решение (ADR-0011)
- Миграция Alembic: добавить `Wallet.icon` (`String`, nullable, дефолт нейтральный — напр. `👛`).
  Только через Alembic, не `CREATE/ALTER` в коде. FK/индексы не нужны.
- Пресет иконок — в конфиге/константах (`config/limits.py` или новый модуль), **минимум по 2** для
  «кошельков» и «карт» (напр. `👛 💰 🏦` и `💳 💵 🪙`); не хардкодить по месту.
- FSM добавления кошелька: после ввода имени — шаг выбора иконки (inline-клавиатура пресета).
  Редактирование кошелька: добавить действие «Сменить иконку».
- Отображение: иконка перед именем кошелька в списках кошельков, в выборе кошелька при создании
  списания, в карточке списания и сводке — через общий форматтер (TASK-0039).
- `repositories/wallets.py`: поддержать `icon` в create/update.

## Критерии приёмки (проверяемые)
- [ ] Миграция добавляет `wallets.icon` (nullable + дефолт); `alembic upgrade head` и автоген-диф
      пустой; миграционный smoke на Postgres зелёный (как TASK-0009).
- [ ] При добавлении кошелька можно выбрать иконку из пресета (≥2 для карт и ≥2 для кошельков);
      при редактировании — сменить.
- [ ] Иконка отображается рядом с именем кошелька в списке кошельков, при выборе кошелька в создании
      списания, в карточке списания и сводке.
- [ ] Существующие кошельки без иконки показываются с дефолтной (не падают).
- [ ] Тексты/пресет — через локали/конфиг, не хардкод в хэндлерах. Доступ к БД — через репозиторий.
- [ ] e2e через `dp.feed_update`: создать кошелёк с иконкой → иконка видна при выборе кошелька и в карточке.
- [ ] Весь CI начисто зелёный.

## Ожидаемые артефакты

  `handlers/wallets.py`, `bot/keyboards.py`, `bot/texts.py`, конфиг пресета, форматтер, тесты.
