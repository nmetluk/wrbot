# TASK-0004: Слой данных справочников + DI сессии БД

## Цель
Заложить слой данных для M2: репозитории кошельков и категорий с изоляцией по `tg_id`
(FR-13), upsert пользователя, лимиты из конфига, и middleware aiogram, выдающий
`AsyncSession` (и репозитории) в хэндлеры. **Без бот-UI** — это TASK-0005.

## Контекст
- ТЗ: §3.3 (кошельки/категории — CRUD), §2 (сущности), §1.3 (изоляция по `tg_id`).
- Требования: FR-8, FR-9, FR-13; NFR-3 (чистый код), NFR-4 (расширяемость).
- ADR: ADR-0003 (SQLAlchemy 2.0 async + Alembic — доступ только через репозитории).
- Текущее состояние: `src/wrbot/db/{models,session}.py` готовы; `repositories/users.py` —
  заглушка; middleware DI ещё нет; хэндлеры не имеют доступа к сессии.
- Правила `CLAUDE.md`: доступ к БД только через репозитории; FK уже в схеме; anti-drift
  (мокать со `spec`/autospec; обращаться к полям ORM напрямую, не через `getattr(...,default)`).

## Критерии приёмки (проверяемые)
- [ ] **Middleware сессии** (`src/wrbot/bot/middlewares/db.py` или аналог): на каждый
      update открывает `AsyncSession` из фабрики, кладёт в `data` (например `session`),
      коммитит при успехе / откатывает при исключении, закрывает. Зарегистрирован в `__main__`/диспетчере.
- [ ] **UserRepository.get_or_create(tg_id)** — создаёт пользователя с дефолтами
      (`notify_time=10:00`, `tz=Europe/Moscow`, `global_days=[5,3,1]`) или возвращает существующего.
- [ ] **WalletRepository**: `create(user_id, name)`, `list_by_user(user_id)` (сортировка по имени/id),
      `get(user_id, wallet_id)`, `rename(user_id, wallet_id, name)`, `delete(user_id, wallet_id)`.
      Все методы фильтруют по `user_id` (изоляция tg_id, FR-13) — чужой объект недоступен.
- [ ] **CategoryRepository**: тот же набор методов с изоляцией по `user_id`.
- [ ] **Лимиты в конфиге** (`config.py`/отдельный модуль лимитов): `MAX_WALLETS`,
      `MAX_CATEGORIES` на пользователя (разумные дефолты, переопределяемые из окружения).
      Сервис/репозиторий не даёт превысить лимит (понятная доменная ошибка/результат).
- [ ] **Валидация имени** (сервис): обрезка пробелов, непустое, ограничение длины;
      запрет дублей имени в пределах пользователя (кошельки/категории отдельно).
- [ ] **Тесты** (через реальную миграцию на временной SQLite, как в `conftest`, `PRAGMA foreign_keys=ON`):
      CRUD каждого репозитория; **изоляция tg_id** (пользователь A не видит/не меняет/не удаляет объекты B);
      `get_or_create` идемпотентен; превышение лимита; дубль имени отклоняется. Моки внутренних
      объектов — со `spec`/autospec.
- [ ] CI зелёный начисто (без `BOT_TOKEN`): ruff, ruff format, mypy, pytest, `alembic upgrade head`, `validate.py`.

## Ожидаемые артефакты

  при наличии списаний станет актуально в M3 — не реализуй преждевременно, но не ломай FK.
- Кроссплатформенно, секреты в `.env`. Перед `done`: отчёт, лог сессии (уникальный ID), state, push.
