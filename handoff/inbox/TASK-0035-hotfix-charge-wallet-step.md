# TASK-0035: HOTFIX — выбор кошелька в создании списания

## Цель
Устранить блокирующее зависание при создании первого (и любого) списания на шаге выбора
кошелька; убрать тупик для нового пользователя (нет кошельков). Выполнять **вне очереди,
до задач M6.** Релиз — патч **v0.1.1**.

## Корневая причина (подтверждено чтением кода)
`handlers/charges_create.py`:
- `process_amount` (`@router.message(NewChargeStates.amount)`) переводит в состояние `wallet`
  и отправляет текст «👛 Загружаю список кошельков…», но **клавиатуру не показывает**.
- Клавиатуру рендерит `show_wallet_selection` (`@router.message(NewChargeStates.wallet)`) —
  он срабатывает только на СЛЕДУЮЩЕЕ сообщение пользователя, которого не будет → бот «висит»
  (**для всех пользователей**, не только новых). У нового пользователя вдобавок 0 кошельков.
- e2e (TASK-0030) это не поймал, т.к. сразу слал callback `charge_wallet_1`, минуя показ клавиатуры.

## Критерии приёмки (проверяемые)
- [ ] **Клавиатура выбора кошелька показывается сразу** после ввода суммы — в `process_amount`
      (дать ему `session: AsyncSession`, загрузить кошельки и отправить клавиатуру одним сообщением),
      без ожидания доп. сообщения. Ошибочный show-хендлер `@router.message(NewChargeStates.wallet)`
      убрать/переработать, чтобы не было «висящего» состояния.
- [ ] **Пустой список кошельков** (например, пользователь удалил все): показывается понятный текст +
      кнопка «➕ Добавить новый кошелёк»; подпоток (`charge_add_wallet` → ввод имени → возврат к выбору
      кошелька с уже созданным) **работает и не зависает**; после создания можно довести списание до конца.
- [ ] **Дефолтный кошелёк для нового пользователя:** `UserRepository.get_or_create` при СОЗДАНИИ нового
      пользователя создаёт кошелёк «Основная карта» (идемпотентно — только при создании, в той же
      транзакции/сессии; не пересоздавать, если пользователь его удалил). Существующие пользователи не дублируются.
- [ ] **Онбординг:** на `/start` — краткая подсказка, что кошельки/карты и категории настраиваются в
      «⚙️ Настройки»; для нового пользователя упомянуть, что создан кошелёк «Основная карта». Тексты — в `texts.py`.
- [ ] **Тесты (e2e через Dispatcher, как в test_e2e_smoke):**
      1) после ввода суммы клавиатура с кошельками приходит **сразу** (без дополнительного сообщения) —
         тест должен падать на текущем коде;
      2) новый пользователь: `/start` → есть «Основная карта» → создание списания доходит до конца
         без ручного создания кошелька;
      3) нет кошельков (удалены) → «➕ Добавить новый кошелёк» в потоке создаёт кошелёк и поток продолжается.
- [ ] Весь CI начисто зелёный (`uv run` ruff/mypy/pytest/alembic/validate).
- [ ] Версия → `0.1.1` (pyproject/`__init__`), запись в корневой `CHANGELOG.md` (Fixed).
