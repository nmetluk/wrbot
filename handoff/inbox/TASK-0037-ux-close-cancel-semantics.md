# TASK-0037: Семантика «Закрыть»/«Отмена» и возврат в меню

## Цель
Сделать выход из экранов логичным и непротиворечивым (отзыв пользователя): на экране-списке
ничего не «отменяется» — «Закрыть» там лишняя и дублирует «◀️ В меню»; «Отмена» уместна только
в диалогах (создание/редактирование), где есть незавершённое действие.

## Контекст
- `keyboards.py`: `get_my_charges_keyboard` (стр. ~304) содержит «❌ Закрыть» (`cancel`) — при этом
  там уже есть «➕ Новое списание» и «◀️ В меню» (TASK-0036). Кнопка дублирующая/сбивающая.
- «❌ Отмена» (`cancel`) используется в FSM-клавиатурах создания/редактирования (кошелёк, категория,
  списание, глобальные уведомления) — там она по делу.

## Рекомендация (реализовать)
- **Убрать «❌ Закрыть»** из `get_my_charges_keyboard` (и из других read-only списков/просмотров,
  если есть) — выход уже обеспечивает «◀️ В меню».
- **«◀️ В меню» (`main_menu`)**: редактирует сообщение в главное меню (не оставляет «висящий» текст).
- **`cancel`** оставить семантикой «отмена диалога»: очистить FSM, показать короткое
  «Действие отменено» и затем **главное меню** (инлайн). Не использовать «Закрыть»/«Отмена»
  на экранах, где нет активного действия.

## Критерии приёмки (проверяемые)
- [ ] В «Мои списания» нет кнопки «Закрыть»; выход — «◀️ В меню» (показывает главное меню),
      есть «➕ Новое списание». Аналогично проверены другие просмотры (карточка и т.п.) — без тупиков и дублей.
- [ ] `cancel` (в диалогах создания/редактирования) сбрасывает FSM, показывает «Действие отменено»
      + главное меню.
- [ ] Нет экранов, где после «закрытия» остаётся «осиротевшее» сообщение без навигации.
- [ ] **E2E через `dp.feed_update`** (по новому чек-листу `executor-guide.md`): открыть «Мои списания»
      → нажать «◀️ В меню» → пришло главное меню (4 кнопки); отмена диалога создания → «Действие
      отменено» + меню. Проверка по реальным апдейтам.
- [ ] Весь CI начисто зелёный.

## Ожидаемые артефакты

- Тесты: дополнения в e2e/handler-тесты.
