# TASK-0034: Ежедневный дашборд (21:00) с графиками

## Цель
В 21:00 (в `ADMIN_TZ`) собирать сводную статистику использования бота, рендерить картинки и
отправлять в админ-канал (ADR-0008). Набор — расширенный (решение владельца).

## Контекст
- ADR-0008 (джоба 21:00 в ADMIN_TZ, отправка фото/альбома), ADR-0010 (audit_log для активности/ошибок).
- Готово: `AdminNotifier` (TASK-0031), `audit_log` (TASK-0032), репозитории.

## Метрики (расширенный набор)
- Рост пользователей (накопительно, 30 дней) — линия.
- Новые/активные пользователи за день (активные = по audit_log).
- Создано списаний за день (14 дней) — бар.
- «Оплачено» за день: количество и сумма — бар/число.
- Распределение активных списаний по периодам (once/monthly/quarterly/yearly) — pie.
- Топ категорий (по числу/сумме активных списаний) — горизонтальный бар.
- Отправлено напоминаний за день; ошибки за день (из audit_log/error-hook).

## Критерии приёмки (проверяемые)
- [ ] `services/stats.py`: функции расчёта всех метрик из БД/`audit_log` (чистые, тестируемые).
- [ ] `services/charts.py`: рендер PNG через matplotlib (**backend `Agg`**, без дисплея), возврат `bytes`.
      Зависимость `matplotlib` добавлена в `pyproject.toml` (+ обновлён `uv.lock` через `uv lock`).
- [ ] **Джоба 21:00** (`CronTrigger`, `timezone=ADMIN_TZ`): собрать метрики → сгенерировать
      картинки → отправить альбомом/по одной в админ-канал с кратким текстовым саммари.
      Ошибка изолирована (не роняет бота), no-op без `ADMIN_CHANNEL_ID`.
- [ ] Тесты: расчёт метрик на засеянной БД (точные числа), `charts.*` возвращают непустой PNG
      (smoke, проверка сигнатуры/размера), джоба собирает и зовёт `send_media_group` (mock Bot).
- [ ] Весь CI начисто зелёный (учесть, что matplotlib не требует дисплея в CI).

## Ожидаемые артефакты

- matplotlib: только `Agg`, не блокировать event loop (рендер — через `asyncio.to_thread` если тяжело).
- Перед `done`: полный CI, отчёт, лог, push.
