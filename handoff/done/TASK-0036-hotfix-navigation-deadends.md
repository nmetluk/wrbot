# TASK-0036: HOTFIX — навигация, устранить тупиковые экраны

## Цель
Убрать тупики в навигации: с любого экрана должен быть выход в главное меню. В первую
очередь — «📋 Мои списания» (сейчас тупик, выйти можно только через `/start`). Релиз v0.1.1.

## Корневая причина (подтверждено чтением кода)
`handlers/charges_list.py` → `list_charges`:
- Пустой список: `edit_text(Texts.my_charges_empty, reply_markup=None)` — **клавиатуры нет вообще**.
- Непустой: `get_my_charges_keyboard` содержит только карточки + «❌ Закрыть» (`cancel`) —
  нет «➕ Новое списание» и «◀️ В меню».
- Обработчик `main_menu` уже существует (`settings.py`) — на него можно вести «В меню».

## Критерии приёмки (проверяемые)
- [ ] **Пустой «Мои списания»:** показывается текст + клавиатура с «➕ Новое списание»
      (`new_charge`) и «◀️ В меню» (`main_menu`). Не `reply_markup=None`.
- [ ] **Непустой «Мои списания»:** в `get_my_charges_keyboard` добавлены «➕ Новое списание»
      (`new_charge`) и «◀️ В меню» (`main_menu`) (помимо карточек).
- [ ] **Нет тупиков:** пройтись по экранам и сообщениям, где `reply_markup=None` или нет пути назад
      (карточка списания, подтверждения, сообщения после действий, экраны ошибок/отмены) — добавить
      возврат в меню/к списку. `grep reply_markup=None` — обоснован для каждого оставшегося случая.
- [ ] **«Отмена»/`cancel` и `main_menu`** надёжно возвращают пользователя в главное меню
      (с инлайн-меню), а не оставляют «голое» сообщение.
- [ ] **Тесты (e2e через Dispatcher):** открыть «Мои списания» (пустой и непустой) → в ответе есть
      кнопки `new_charge` и `main_menu`; нажатие `main_menu` показывает главное меню (4 кнопки).
      Тесты должны падать на текущем коде (пустой экран без кнопок).
- [ ] Весь CI начисто зелёный (`uv run` ruff/mypy/pytest/alembic/validate).
