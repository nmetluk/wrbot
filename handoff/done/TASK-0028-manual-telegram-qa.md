# TASK-0028: Ручная проверка в живом Telegram

## Цель
Подтвердить работоспособность v1 в реальном Telegram перед боевым деплоем. Юнит-тесты
зелёные, но UX-баги и интеграционные эффекты (как тихая потеря данных из TASK-0027)
надёжно ловятся только живым прогоном. Это финальный gate перед релизом.

## Подготовка (исполнитель/владелец)
- Создать тестового бота у @BotFather, получить `BOT_TOKEN` (в `.env`, не в репозиторий).
- Поднять локально (`python -m wrbot`) или через `docker compose up -d` (SQLite-профиль).
- Для проверки уведомлений: создать списание с `next_date` так, чтобы напоминание попало
  на ближайшую минуту, и/или временно выставить `notify_time` на «сейчас» в нужном tz.

## Чеклист (отметить ✅/❌, приложить замечания) — выполняет владелец/тестировщик
- [ ] `/start` → главное меню (4 кнопки). `/help` → справка. `/cancel` вне диалога — не ломает.
- [ ] **Кошельки:** добавить (имя сохранилось ПОСЛЕ перезахода в меню!), переименовать, удалить.
- [ ] **Категории:** добавить/переименовать/удалить (сохранение проверить перезаходом).
- [ ] **Создание списания:** пройти все шаги (название→сумма→кошелёк[+добавить новый]→категория[или пропуск]
      →дата ДД.ММ.ГГГГ→период→уведомления). **Списание реально появляется в «Мои списания».**
- [ ] **Мои списания:** список по дате; карточка; редактирование (изменение сохраняется);
      удаление с подтверждением.
- [ ] **«Оплачено»:** периодическое — дата сдвинулась на период; одноразовое — ушло из списка.
- [ ] **Глобальные уведомления (FR-10):** сменить время и дни — **значения сохраняются** (перезайти и проверить).
- [ ] **Часовой пояс:** сменить — сохраняется; напоминание приходит в локальное `notify_time`.
- [ ] **Уведомление приходит** в назначенное время; кнопки «Оплачено»/«Отложить»/«Редактировать» работают;
      «Отложить» не двигает дату платежа, напоминает повторно.
- [ ] **Изоляция:** со второго аккаунта не видно чужих данных.
- [ ] **Удаление кошелька с активными списаниями** — понятное сообщение, не падение.
- [ ] **Рестарт процесса** в день уведомления — напоминание не задваивается.

## Критерии приёмки
- [ ] Чеклист пройден; результаты + найденные дефекты зафиксированы в
      `handoff/reports/QA-MANUAL-<дата>.md`.
- [ ] Критичные/функциональные дефекты — заведены отдельными задачами; при их отсутствии —
      проект готов к релизу (тег версии + корневой CHANGELOG).

## Ограничения / заметки
- Это ручная проверка (не автотест); секреты — только в `.env`. Результат — отчёт в `handoff/reports/`.
