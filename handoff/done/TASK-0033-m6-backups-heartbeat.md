# TASK-0033: Ежечасный бэкап + статус + сводка

## Цель
Автоматические ежечасные бэкапы БД с ротацией и ежечасное сообщение в админ-канал:
бэкап ОК + «бот работает» + краткая сводка (ADR-0009, ADR-0008).

## Контекст
- ADR-0009 (метод по типу БД, ротация 36, статус в канал, том `./backups`), ADR-0008 (канал/джобы).
- Готово: `AdminNotifier` (TASK-0031), планировщик (`add_job`), репозитории.

## Критерии приёмки (проверяемые)
- [ ] `services/backup.py`: создание бэкапа в `./backups` по типу БД (детект из `DATABASE_URL`):
      SQLite → `VACUUM INTO 'backups/wrbot-YYYYMMDD-HHMM.sqlite3'`; PostgreSQL → `pg_dump` →
      `backups/wrbot-YYYYMMDD-HHMM.sql.gz`. Имя файла с UTC-таймстампом.
- [ ] **Ротация:** хранить 36 последних, старшие удалять (детерминированно, тест).
- [ ] **Ежечасная джоба** (APScheduler, `register_*` в `scheduler/app.py`): выполнить бэкап →
      отправить в админ-канал сообщение: статус бэкапа (✅/❌ + файл + размер), heartbeat
      («бот работает»), **краткая сводка**: всего пользователей, всего активных списаний,
      создано списаний за час, отмечено оплат за час. Ошибка бэкапа → ❌-сообщение, джоба не падает.
- [ ] **Docker:** том `./backups:/app/backups` в `docker-compose.yml`; для Postgres-пути —
      `postgresql-client` в `Dockerfile` (pg_dump). Каталог доступен non-root пользователю.
- [ ] Тесты: SQLite-бэкап создаёт валидный файл (можно открыть/прочитать), ротация оставляет 36,
      сводка считается верно на засеянной БД, сообщение собрано (mock Bot), ошибка изолирована.
- [ ] Весь CI начисто зелёный.

## Ожидаемые артефакты
- Код: `src/wrbot/services/backup.py`, `services/stats.py` (или часть — краткая сводка),
  `scheduler/app.py` (+ джоба), правки `docker-compose.yml`, `Dockerfile`, `.dockerignore` (не игнорировать backups том), `.gitignore` (игнорировать `backups/`).
- Тесты: `tests/test_backup.py`.

## Ограничения / заметки
- Бэкап-файлы в канал НЕ выгружаем (решение владельца) — только статус. Секреты/креды в сообщения не попадают.
- `backups/` — в `.gitignore` (не коммитим дампы). Перед `done`: полный CI, отчёт, лог, push.
