# TASK-0015: Уведомление и его кнопки

## Цель
Реализовать «принимающую» сторону напоминания: формат сообщения и обработку кнопок
«Оплачено» / «Отложить» / «Редактировать» (FR-12). **Без свипа/планировщика** (TASK-0016).

## Контекст
- ТЗ §3.4 (в уведомлении: название, сумма, кошелёк, дата + кнопки Оплачено/Отложить/Редактировать).
- Требования: FR-12, FR-7, FR-13, FR-14.
- ADR-0005 («Отложить» = повтор на следующий день, `next_date` не меняется),
  ADR-0006 (`mark_paid` уже есть в `ChargeRepository`).
- Готово: `ChargeRepository.mark_paid` (TASK-0010), `ChargeRepository.snooze` (TASK-0014),
  FSM редактирования (TASK-0012). Уроки: специфичные callback-фильтры + router-тест (TASK-0008),
  тексты только в `texts.py`, моки autospec, реальный рендер.

## Критерии приёмки (проверяемые)
- [ ] **Текст уведомления** (`texts.py`, шаблон с подстановками): название, сумма, кошелёк, дата.
- [ ] **Inline-клавиатура** с тремя кнопками; callback'и специфичны:
      `remind_paid_<id>`, `remind_snooze_<id>`, `remind_edit_<id>`.
- [ ] **«Оплачено»** → `ChargeRepository.mark_paid` (сдвиг/закрытие), пользователю — подтверждение
      (для периодического показать новую дату). Изоляция по `tg_id`.
- [ ] **«Отложить»** → `ChargeRepository.snooze(..., until=завтра)` — `next_date` не меняется;
      сообщить, что напомним завтра (ADR-0005).
- [ ] **«Редактировать»** → вход в FSM редактирования (переиспуя TASK-0012).
- [ ] Тексты только в `texts.py`; ≥1 тест реального рендера.
- [ ] **Тесты:** хэндлеры (autospec) — paid (периодич./once), snooze (ставит завтра, не трогает next_date),
      edit-вход; **router-тест** для `remind_*` (через зарегистрированные фильтры).
- [ ] Весь CI начисто без BOT_TOKEN зелёный (ruff format/check, mypy, pytest, alembic, validate).
