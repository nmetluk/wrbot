# TASK-0040: Поменять шаги (период → дата) и валидировать дату относительно периода

## Цель (отзыв пользователей, п.2)
Дать осмысленную проверку даты: следующая дата регулярного платежа не может быть дальше одного
периода от сегодня. Чтобы это проверить, нужно знать период **до** ввода даты → поменять шаги.

## Решение владельца (зафиксировано)
**Окно периода (верхняя граница):** дата > сегодня И дата ≤ сегодня + период:
- `once` (одноразово): дата > сегодня (без верхней границы — разовый платёж может быть когда угодно в будущем).
- `monthly`: сегодня < дата ≤ сегодня + 1 месяц.
- `quarterly`: сегодня < дата ≤ сегодня + 3 месяца.
- `yearly`: сегодня < дата ≤ сегодня + 1 год.
Перенос конца месяца — как в существующей доменной логике дат (ADR-0006); переиспользовать её,
не изобретать заново.

## Изменения
- В `NewChargeStates` поменять порядок: **period → next_date** (сейчас `next_date`(5) → `period`(6)).
- `handlers/charges_create.py`: сначала шаг периода (клавиатура), затем ввод даты; валидацию даты
  вынести в `services/charges.py` или `services/dates.py` как `validate_next_date(period, d, today)`
  (чистая функция, покрыта unit-тестами по всем периодам и границам).
- Тексты ошибок (`new_charge_invalid_date`) — обновить под новое правило (через локали), сообщать
  допустимое окно (напр. «для ежемесячного — не позже ДД.ММ.ГГГГ»).
- Учесть редактирование (TASK-0045 переиспользует этот же flow) — валидатор общий.

## Критерии приёмки (проверяемые)
- [ ] Шаги: …→ категория → **период** → **дата** → уведомления. (Сейчас дата шла перед периодом.)
- [ ] `validate_next_date` покрыт unit-тестами: для каждого периода — граничные случаи
      (сегодня → отклонить; +период → принять; +период+1 день → отклонить для monthly/quarterly/yearly;
      для `once` верхней границы нет), перенос конца месяца (31 → последний день короткого месяца).
- [ ] Ввод даты вне окна → понятная ошибка с допустимым диапазоном (через локаль), шаг не проходит.
- [ ] e2e через `dp.feed_update`: выбрать ежемесячно → дата на 2 месяца вперёд отклонена,
      дата в пределах месяца принята → дошли до сводки.
- [ ] Весь CI начисто зелёный.
