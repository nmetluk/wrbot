# TASK-0041: Группировка «Мои списания» по категориям

## Цель (отзыв пользователей, п.3)
Сделать «Мои списания» обзорными: краткий текст с группировкой по категориям и ближайшей датой,
а под сообщением — кнопки перехода в категорию; после кнопок категорий — кнопки списаний без категории.

## Желаемый вид (из отзыва)
- Текстовое сообщение: по категориям, в каждой — строки вида `Списание1 (666 ₽) — 23.06.2026`
  (ближайшая дата списания, формат ДД.ММ.ГГГГ через форматтер TASK-0039).
- Инлайн-кнопки: сначала по кнопке на **категорию** (переход в список списаний этой категории),
  затем — кнопки **списаний без категории** (прямо на карточку).

## Решение
- `repositories/charges.py`: метод выборки активных списаний пользователя, сгруппированных по
  категории (или сортировка + группировка в сервисе); внутри группы — по ближайшей дате.
- `services/`: подготовка структуры для рендера (категория → список; отдельная «без категории»).
- `handlers/charges_list.py`: новый рендер `list_charges` (краткий текст + клавиатура категорий).
- Новый callback вида `charges_cat_<id>` → список списаний категории (карточки/кнопки),
  переиспользует существующий просмотр карточки (`charge_item_`); «без категории» — `charges_cat_none`
  или прямые `charge_item_` кнопки.
- `keyboards.py`: клавиатура категорий + кнопки списаний без категории + «➕ Новое списание»/«◀️ В меню»
  (без тупиков, по чек-листу executor-guide).

## Критерии приёмки (проверяемые)
- [ ] «Мои списания»: текст сгруппирован по категориям, в каждой строке — название, сумма, ближайшая
      дата ДД.ММ.ГГГГ; есть блок/кнопки для списаний без категории.
- [ ] Кнопки категорий ведут в список списаний этой категории; кнопки списаний без категории —
      на карточку. Везде есть выход (нет тупиков).
- [ ] Пустой случай (нет списаний) сохраняет текущее поведение (`my_charges_empty` + кнопки).
- [ ] Все тексты — через локали; даты/суммы — через форматтер (TASK-0039).
- [ ] e2e через `dp.feed_update`: список → переход в категорию → карточка → возврат в меню;
      сценарий со списанием без категории.
- [ ] Весь CI начисто зелёный.

## Ожидаемые артефакты
- `repositories/charges.py`, `handlers/charges_list.py`, `bot/keyboards.py`, `bot/texts.py`, тесты.

## Ограничения / заметки
- Доступ к БД — только через репозитории. Группировку лучше готовить в сервисе/репозитории, не в хэндлере.
- Перед `done`: полный CI, отчёт, лог, push.
