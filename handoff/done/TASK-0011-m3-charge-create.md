# TASK-0011: Пошаговое создание списания

## Цель
Реализовать «➕ Новое списание» — пошаговый диалог создания списания поверх
ChargeRepository (FR-3, FR-4, FR-11 на этапе создания).

## Контекст
- ТЗ §3.2 (создание: название → сумма → кошелёк(+добавить новый) → категория(или пропуск)
  → дата → периодичность → уведомления), §3.1.
- Требования: FR-3, FR-4, FR-11, FR-13, FR-14 (русский).
- Готово: `ChargeRepository`/валидация (TASK-0010); `start.py` имеет заглушку `new_charge`
  (заменить); репозитории кошельков/категорий (M2); DI-сессия через middleware.
- Уроки: тонкие хэндлеры; тексты только в `texts.py`; **специфичные callback-фильтры**
  и тест через роутер (урок TASK-0008); моки со `spec`/autospec; реальный рендер текста.

## Критерии приёмки (проверяемые)
- [ ] **FSM `NewChargeStates`** (`states.py`): name → amount → wallet → category → next_date → period → notify.
- [ ] Шаги диалога:
      1) Название (валидация непустое/длина);
      2) Сумма (Decimal, >0, понятная ошибка при неверном вводе — через `services/charges`);
      3) Кошелёк: выбор из существующих + кнопка «➕ Добавить новый» (переход к созданию кошелька,
         затем возврат в поток); если кошельков нет — предложить создать;
      4) Категория: выбор или «Пропустить» (`category_id=null`);
      5) Дата следующего списания: ввод в формате `ДД.ММ.ГГГГ`, валидация (корректная дата,
         не в прошлом — иначе понятное сообщение);
      6) Периодичность: одноразово/ежемесячно/ежеквартально/ежегодно (FR-4);
      7) Уведомления (FR-11): «глобальные» (`individual_days=null`) / «свои» (ввод дней, валидация) /
         «отключить» (`individual_days=[]`).
- [ ] Сохранение через `ChargeRepository.create`; `get_or_create` пользователя; изоляция по tg_id.
- [ ] `/cancel` и кнопка «Отмена» прерывают диалог и чистят FSM на любом шаге.
- [ ] Тексты — только в `texts.py` (включая шаблоны с подстановками); ≥1 тест реального рендера.
- [ ] **Тесты:** шаги/ветки FSM (моки репозиториев autospec), валидация суммы/даты/дней,
      «добавить новый кошелёк» внутри потока, отмена; callback-роутинг — специфичные фильтры
      + тест через роутер. Сохранить зелёный pytest.
- [ ] Весь CI начисто без BOT_TOKEN зелёный (ruff format/check, mypy, pytest, alembic, validate).
