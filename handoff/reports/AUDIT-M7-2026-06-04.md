## По задачам
- **0039 (фикс отображения)** — принято ранее (AUDIT-M7-interim). Общий `formatters.py`, ДД.ММ.ГГГГ,
  реальные имена, хардкоды устранены.
- **0040 (период→дата + окно)** — принято ранее. `validate_next_date`/`get_period_upper_bound` на
  clamp ADR-0006; побочно фикс коллизии роутинга confirm_create/confirm_delete.
- **0041 («Мои списания» группировка)** — принято ранее. `list_active_grouped_by_category` + навигация.
- **0042 (иконки кошельков, ADR-0011).** Миграция `wallets.icon` (`String(10)`, NOT NULL,
  **`server_default="👛"` + явный backfill** — безопасно на непустой таблице). Пресет
  `WALLET_ICONS` в `config.py` (6 эмодзи). `WalletRepository.create(..., icon)` + `set_icon`
  (с аудитом). FSM выбора иконки при добавлении + «Сменить иконку» при редактировании. Иконка
  показывается через `resolve_wallet_name` → «{icon} {name}» во всех местах. Дефолт «Основная карта» = «💳».
- **0043 (схема целей + UI, ADR-0012).** Миграция `categories.notify_chat_ids` (Text, nullable).
  `validators.is_valid_chat_id` (только числовой, диапазон). Repo `get/add/remove_notify_chat_id`
  (JSON, **дедуп `sorted(set(...))`**, пусто→NULL). UI в настройках категории (список/добавить/удалить + FSM).
- **0044 (дубли в каналы + уведомление владельца, ADR-0012).** В `scheduler/sweep.py`: личное
  напоминание фиксируется `sent_repo.record` **до** дублей → **идемпотентность только на personal**.
  Дубли в общем try/except (изоляция), по-чатовый `TelegramForbiddenError`/`TelegramBadRequest`
  → уведомление владельцу с chat_id (без чувствительного), прочие ошибки логируются и не ломают
  остальные чаты/личное. Тесты с mock `Bot` (multi + forbidden).
- **0045 (живая карточка редактирования).** `start_edit` грузит данные в FSM, хранит
  `edit_card_msg_id`/`edit_chat_id`, рендерит живую карточку `build_edit_live_card` (форматтер 0039);
  на шагах обновляет то же сообщение (нет осиротевших черновиков). Cancel → «Действие отменено» + меню
  без сохранения; save — ветка update. e2e на изменение суммы + cancel.
