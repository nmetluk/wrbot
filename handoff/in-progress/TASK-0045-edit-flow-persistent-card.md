# TASK-0045: Переработка UX редактирования списания

## Цель (отзыв пользователей, п.6)
Сейчас при редактировании информация о редактируемом списании пропадает. Нужно: карточка с
текущими данными списания **висит и обновляется на каждом шаге**, отражая подтверждённые изменения;
а уже введённые/черновые значения — не засоряют рабочую область.

## Контекст (по коду)
- `handlers/charges_create.py:start_edit_charge` загружает поля в FSM (`NewChargeStates`) и сразу
  переходит к шагу «Название», показывая только `charge_edit_started` + приглашение — **без сводки
  текущих данных**. Далее прогоняется обычный flow создания, где промежуточная карточка не видна.

## Решение
- При старте редактирования и на каждом шаге показывать **актуальную карточку** списания
  (через форматтер TASK-0039): какие значения сейчас (исходные + уже изменённые в этой сессии),
  и какой шаг редактируется. Обновлять это сообщение по мере подтверждения шагов
  (`edit_text` существующего сообщения, без накопления «простыни»).
- Введённые на шаге данные не должны «зависать» отдельными сообщениями-черновиками — рабочая
  область чистая: одно живое сообщение-карточка + приглашение к текущему шагу.
- Согласовать с новым порядком шагов (период → дата, TASK-0040) и общим форматтером (TASK-0039).
- Сохранение — существующий `confirm_create_charge` (ветка `editing_charge_id`).

## Критерии приёмки (проверяемые)
- [ ] При входе в редактирование видна карточка текущих данных списания (имя, сумма, кошелёк,
      категория, дата ДД.ММ.ГГГГ, период, уведомления) — через форматтер.
- [ ] На каждом шаге карточка обновляется, отражая уже принятые изменения; нет «осиротевших»
      промежуточных сообщений и дублей.
- [ ] Отмена редактирования (`cancel`) → «Действие отменено» + меню (как TASK-0037), данные не сохранены.
- [ ] Сохранение применяет изменения (ветка update), показывает `charge_edit_saved` + меню.
- [ ] e2e через `dp.feed_update`: открыть редактирование → изменить сумму → карточка обновилась →
      сохранить → в карточке списания новая сумма (персистентность в БД).
- [ ] Весь CI начисто зелёный.
