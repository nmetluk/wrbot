# state/ — машиночитаемое состояние проекта

Назначение каталога — дать любому новому агенту или человеку **мгновенно**
восстановить контекст без чтения всей истории. Это «снимок правды», который
обновляется в конце каждой сессии.

| Файл | Формат | Содержит |
|------|--------|----------|
| `project.json` | JSON | Канонический снимок: текущая фаза/майлстоун, активная задача, последняя сессия, ключевые факты, стек. |
| `backlog.json` | JSON | Индекс всех задач и их статусов, собирается из `handoff/` скриптом `scripts/update_state.py`. |
| `CHANGELOG.md` | Markdown | Человекочитаемая хронология значимых изменений (Keep a Changelog). |

## Правила
- `project.json` — единственный «ручной» (обновляется агентом) файл; держи его в актуальном виде.
- `backlog.json` — **генерируемый**, не редактируй руками: `python scripts/update_state.py`.
- CI проверяет, что `backlog.json` соответствует фактическому содержимому `handoff/`.
- Все даты — в ISO-8601 UTC (`YYYY-MM-DDTHH:MM:SSZ`).
