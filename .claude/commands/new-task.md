---
description: Создать новую задачу в handoff/inbox из шаблона (роль архитектора)
---

Ты — архитектор. Создай хорошо поставленную задачу:

1. `git pull --ff-only`.
2. Уточни у пользователя (или из roadmap) цель, майлстоун и зависимости, если неясно.
3. Создай файл: `python scripts/new_task.py --title "<заголовок>" --slug <латиницей>
   --milestone <Mx> [--depends-on TASK-0001,...]`.
4. Открой созданный файл в `handoff/inbox/` и заполни:
   - **Цель** (1–3 предложения, зачем).
   - **Контекст**: ссылки на ТЗ §, FR/NFR ID, нужные ADR, файлы.
   - **Критерии приёмки** — конкретные и проверяемые.
   - **Ожидаемые артефакты** (код, тесты, обновления доков).
   - **Ограничения** и **зависимости**.
   Задача должна быть выполнима за одну сессию исполнителя; иначе разбей на несколько.
5. `python scripts/update_state.py` — обнови backlog.
6. Заверши через `/end-session` (роль architect): лог сессии, обновление
   `state/project.json` (`next_step`), commit, push.
