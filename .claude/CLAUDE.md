# CLAUDE.md — инструкции для Claude Code в этом репозитории

> **Сначала прочитай [`../AGENTS.md`](../AGENTS.md)** — это главный операционный
> мануал (роли, ритуал сессии, контракт handoff, правила). Этот файл — точка входа
> именно для Claude Code и краткая шпаргалка.

## Ритуал начала ЛЮБОЙ сессии
1. `git pull` — GitHub (`origin/main`) единственный источник правды.
2. Прочитай `AGENTS.md`, затем `state/project.json` и последний файл в `sessions/`.
3. Определи роль и открой гайд: `docs/workflow/executor-guide.md` (обычно ты — исполнитель)
   или `docs/workflow/architect-guide.md`.

## Слэш-команды
- `/take-task` — взять верхнюю задачу из `handoff/inbox/` и начать работу.
- `/new-task` — (архитектор) создать новую задачу в `inbox/`.
- `/end-session` — завершить: отчёт, перенос задачи, лог сессии, обновление state, push.
- `/sync` — синхронизироваться с `origin/main` и пересобрать состояние.
- `/audit` — провести аудит майлстоуна (отдельной сессией).

## Незыблемые правила
- **Источник правды — GitHub.** Любое изменение завершается `git push`.
- **Секреты — только в `.env`** (в `.gitignore`). Токен бота, PAT, строку БД не коммить.
- **Кроссплатформенность.** Скрипты — Python (`scripts/`), пути — `pathlib`, переносы — LF.
- **Один фокус на задачу.** Смежное — новой задачей в `inbox/`, не раздувай scope.
- **Сомнение в требовании → `handoff/blocked/`**, не угадывай.
- **Каждый шаг — отдельная сессия.** Перед завершением убедись, что контекст полностью
  сохранён в репозитории (см. `docs/workflow/context-preservation.md`).
- **Conventional Commits** со ссылкой `Task: TASK-XXXX` (см. `CONTRIBUTING.md`).
