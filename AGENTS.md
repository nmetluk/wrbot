# AGENTS.md — операционный мануал для ИИ-агентов

> Это первый файл, который читает **любой** агент (архитектор или исполнитель) в
> начале **каждой** сессии. Он — точка входа в контекст. Стандарт `AGENTS.md`
> поддерживается разными ИИ-инструментами; для Claude Code дополнительно см.
> [`.claude/CLAUDE.md`](.claude/CLAUDE.md), который ссылается сюда.

## 0. Ритуал начала сессии (ОБЯЗАТЕЛЬНО)
Выполни по порядку, прежде чем что-либо делать:
1. `git pull` (или убедись, что рабочая копия = `origin/main`). GitHub — источник правды.
2. Прочитай этот файл (`AGENTS.md`).
3. Прочитай `state/project.json` — текущая фаза, последняя сессия, открытые задачи.
4. Прочитай последний файл в `sessions/` — чем закончилась прошлая сессия.
5. Определи свою роль (архитектор или исполнитель) и открой соответствующий гайд:
   - `docs/workflow/architect-guide.md`
   - `docs/workflow/executor-guide.md`

## 1. Роли
- **Архитектор** работает в Cowork. Ставит задачи (`/new-task` → `handoff/inbox/`),
  ревьюит отчёты (`handoff/reports/`), принимает/отклоняет работу, инициирует аудиты.
  **Архитектор не пишет код продукта** — он проектирует и контролирует.
- **Исполнитель** работает в Claude Code на машине с PAT. Берёт задачу
  (`/take-task`), реализует её в `src/`, пишет отчёт, коммитит и пушит. Завершает
  сессию через `/end-session`. **Исполнитель не меняет ТЗ и архитектурные решения**
  без задачи — при сомнении переводит задачу в `handoff/blocked/`.

## 2. Контракт handoff (кратко)
Задача проходит по конвейеру каталогов:
`inbox/` → `in-progress/` → (`done/` | `blocked/`). Подробно:
`docs/workflow/handoff-protocol.md`.
- Файл задачи: `TASK-XXXX-краткое-имя.md` по шаблону `docs/templates/task.template.md`.
- Отчёт исполнителя: `handoff/reports/TASK-XXXX-report.md` по шаблону
  `docs/templates/report.template.md`.
- Состояние конвейера дублируется в `state/backlog.json` (генерируется
  `scripts/update_state.py`, проверяется CI).

## 3. Сохранение контекста между сессиями (золотое правило)
Сессия эфемерна — репозиторий вечен. Перед завершением сессии в репозитории
должно остаться достаточно, чтобы **другой агент или человек без доступа к чату**
полностью восстановил контекст:
- Лог сессии в `sessions/SESSION-YYYY-MM-DD-NN.md` (шаблон в `docs/templates/`).
- Обновлённый `state/project.json` и `state/CHANGELOG.md`.
- Отчёт по задаче в `handoff/reports/`.
- Осмысленные сообщения коммитов (Conventional Commits, см. `CONTRIBUTING.md`).

## 4. Жёсткие правила
- **Секреты только в `.env`** (в `.gitignore`). PAT, токен бота — НИКОГДА в репозиторий.
  Если секрет случайно попал в историю — это инцидент безопасности, см. `docs/workflow/audit-protocol.md`.
- **Кроссплатформенность.** Никаких bash-only команд в скриптах; автоматизация — на
  Python (`scripts/`). Пути — через `pathlib`. Перенос строк — LF.
- **Без ручных шагов.** Если действие повторяется — оформи скрипт или слэш-команду.
- **Одна задача — один фокус.** Не расширяй scope задачи; новое — в `handoff/inbox/` отдельной задачей.
- **Аудит после крупного раздела.** По завершении майлстоуна (`docs/roadmap.md`) —
  отдельная сессия-аудит (безопасность, архитектура, качество).
