# Компоненты и предлагаемая структура `src/`

> Предложение, финализируется в TASK-0001. Цель — чистые слои и расширяемость (NFR-4).

```
src/
└── wrbot/
    ├── __init__.py
    ├── __main__.py          # точка входа: запуск бота + планировщика
    ├── config.py            # загрузка настроек из окружения (.env): токен, БД, TZ
    ├── logging.py           # настройка логирования (структурно, в logs/ — gitignored)
    ├── bot/
    │   ├── handlers/        # хэндлеры по доменам: start, charges, wallets, categories, settings
    │   ├── keyboards/       # сборка inline-клавиатур
    │   ├── states.py        # FSM-состояния пошаговых диалогов
    │   └── texts.py         # русские тексты (единое место, готово к i18n)
    ├── services/            # бизнес-логика: даты/периоды, расчёт напоминаний, «оплачено/отложить»
    ├── repositories/        # доступ к данным по сущностям (изоляция по tg_id)
    ├── db/                  # подключение, схема, миграции
    ├── scheduler/           # интеграция APScheduler, постановка/снятие джоб напоминаний
    └── models.py            # доменные модели/типы (dataclass/pydantic)
tests/                       # pytest: unit (сервисы/даты) + интеграционные (репозитории)
```

## Ответственности
| Компонент | Отвечает за | Ключевые требования |
|-----------|-------------|---------------------|
| `config` | Секреты/настройки из окружения | NFR-6 |
| `bot/handlers` | Команды, кнопки, пошаговые диалоги | FR-1…FR-12 |
| `bot/keyboards`, `texts` | Inline-UI, русские тексты | FR-1, FR-14 |
| `services` | Период→дата, кому/когда напоминать, оплачено/отложить | FR-7, FR-11, FR-12 |
| `repositories` | CRUD по `tg_id` | FR-5…FR-9, FR-13 |
| `db` | Схема, миграции, SQLite→PG | NFR-5 |
| `scheduler` | Своевременные напоминания ±1 мин, устойчивость к рестарту | NFR-1, NFR-2 |

Каждый блок разрабатывается отдельной задачей из `handoff/inbox`, см. `docs/roadmap.md`.
