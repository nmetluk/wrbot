# ADR-0008: Админ-канал и подсистема наблюдаемости

- **Статус:** Accepted
- **Дата:** 2026-06-03
- **Контекст сессии:** SESSION-2026-06-03-03
- **Связанные задачи:** TASK-0031…0034 (M6)

## Контекст
После релиза v0.1.0 нужна эксплуатационная наблюдаемость: подтверждение, что бот жив и
бэкапы проходят; краткая статистика; ежедневный дашборд. Нужен единый приватный канал для
админа (по образцу admin alerts в соседнем `whois-watcher`).

## Решение
- Единый **приватный админ-канал**; его ID — в `ADMIN_CHANNEL_ID` (env, int). Если не задан —
  все админ-функции работают как **no-op** (бот функционирует без админ-канала; тесты/локалка не требуют канала).
- Отдельный сервис `services/admin_notify.py` поверх `aiogram.Bot`: `send_text`, `send_photo`/`send_media_group`;
  **изоляция ошибок** (сбой отправки в канал не роняет бота/джобу), без `print`, через logging.
- **Санитизация:** в админ-сообщения НИКОГДА не попадают `BOT_TOKEN`, строка `DATABASE_URL` с
  кредами, секреты. (IP сервера — не логируем; при необходимости — отдельным явным конфигом.)
- Периодические задачи — на том же `AsyncIOScheduler` (UTC внутри, человекочитаемое время в
  пользовательском поясе): **ежечасно** — бэкап + статус/heartbeat + краткая сводка (ADR-0009);
  **ежедневно 21:00** (в `ADMIN_TZ`, дефолт Europe/Moscow) — дашборд с картинками (TASK-0034).
- Аудит-лог — ADR-0010.

## Последствия
- **+** Прозрачность эксплуатации без внешних систем; всё в одном канале.
- **+** Безопасно отключаемо (нет канала → no-op); не влияет на основной функционал.
- **−** Дополнительные периодические джобы и зависимость для графиков (matplotlib, TASK-0034).
- Время дашборда (21:00) считается в `ADMIN_TZ`; триггер регистрируется с этим поясом.
