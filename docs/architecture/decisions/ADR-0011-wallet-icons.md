# ADR-0011: Иконки кошельков/карт — эмодзи из пресета

- **Статус:** Accepted
- **Дата:** 2026-06-03
- **Контекст сессии:** SESSION-2026-06-03-16
- **Связанные задачи:** TASK-0042

## Контекст
Отзыв пользователей: кошельки/карты визуально неразличимы. Нужна возможность присвоить
кошельку иконку, чтобы отличать карты в списках, карточке списания и сводке создания.
Ограничения: интерфейс — текстовые Telegram-сообщения (не веб); хранилище простое
(SQLite/PostgreSQL); не усложнять инфраструктуру.

## Рассмотренные варианты
1. **Эмодзи из готового пресета.** Плюсы: рендерится прямо в тексте сообщений и в подписях
   inline-кнопок, нулевая инфраструктура (хранится как короткая строка), кроссплатформенно.
   Минусы: ограниченный набор, не кастомные картинки.
2. **Загрузка пользовательских изображений (Telegram `file_id`).** Плюсы: полная кастомизация.
   Минусы: нельзя показать внутри текстовой строки списка; нужно хранение/валидация/лимиты
   `file_id`, обработка протухания; заметно сложнее. Избыточно для задачи «различать карты».

## Решение
Вариант 1 — **эмодзи из фиксированного пресета** (решение владельца). Колонка `Wallet.icon`
(`String`, nullable, дефолт — нейтральный кошелёк, напр. `👛`). Пресет — минимум по 2 иконки
для «кошельков» и «карт» (напр. `👛 💰 🏦` и `💳 💵 🪙`), вынесен в конфиг/константы, не хардкод
по месту. Выбор иконки — inline-клавиатура при добавлении и редактировании кошелька.
Иконка показывается рядом с названием кошелька везде, где он отображается (списки, карточка,
сводка), через общий форматтер (TASK-0039).

## Последствия
- Положительные: визуальное различение без файлового хранилища; иконка живёт в тексте сообщений
  и подписях кнопок; тривиальная миграция (одна nullable-колонка с дефолтом).
- Отрицательные / компромиссы: набор иконок ограничен пресетом; смена пресета потом — отдельная задача.
- Разблокирует: TASK-0042 (иконки в add/edit + отображение). Зависит от общего форматтера (TASK-0039).
