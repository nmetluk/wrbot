# ADR-0013: Валюты списаний — bundled ISO 4217, валюта на списание, пресеты + список/поиск

- **Статус:** Accepted
- **Дата:** 2026-06-04
- **Контекст сессии:** SESSION-2026-06-04-11
- **Связанные задачи:** TASK-0049 (схема + справочник + отображение), TASK-0050 (выбор валюты в потоке)

## Контекст
Срочный отзыв: списания должны поддерживать валюты. Сейчас сумма везде печатается с зашитым «₽»
(`texts.py`, `charges_list.py`), у `Charge` нет валюты. Нужно: пресеты RUB/EUR/USD/KZT/KGS,
полный справочник кодов валют в проекте (оффлайн), механизм выбора любой другой валюты.
Ограничения проекта: оффлайн (без сетевых вызовов в рантайме), миграции только через Alembic,
тексты через локали, доступ к БД через репозитории.

## Решения владельца (зафиксированы)
- Механизм для «другой» валюты — **список + поиск** (постраничный список всех валют + ввод
  кода/названия для фильтра).
- Валюта по умолчанию — **запоминаем последнюю** использованную пользователем (старт — RUB).

## Рассмотренные варианты (хранение справочника)
1. **Bundled JSON в пакете** (`src/wrbot/data/currencies.json`), сгенерирован из ISO 4217. Плюсы:
   полностью оффлайн, версионируется в git, грузится один раз при старте. Минусы: обновлять вручную
   при изменениях ISO (редко). **Выбрано.**
2. Тянуть из сети/таблицы курсов. Минусы: сетевой рантайм-вызов, против оффлайн-инвариантов; курсы
   не нужны (конвертация не требуется — только хранение валюты суммы). Отклонено.

## Решение
- **Справочник в проекте:** `src/wrbot/data/iso4217.csv` (сырьё ISO 4217, источник
  datasets/currency-codes) + сгенерированный `src/wrbot/data/currencies.json`: активные валюты
  (без WithdrawalDate, без неденежных кодов XXX/XTS/металлов/фондов), поля `code`/`name`/`minor`
  (+`symbol` для основных), а также `presets=[RUB,EUR,USD,KZT,KGS]` и `default=RUB`. 153 валюты.
- **Хранение валюты — на списании:** колонка `Charge.currency` (`String(3)`, NOT NULL, дефолт `RUB`,
  backfill существующих в RUB). Конвертация сумм НЕ делается — валюта лишь атрибут суммы.
- **Запоминание последней:** колонка `User.last_currency` (`String(3)`, nullable/дефолт RUB);
  обновляется при создании списания; подставляется как преселект в потоке создания.
- **Отображение:** единый форматтер суммы (расширение `services/formatters.py` из TASK-0039) —
  `{amount} {symbol|code}`; зашитые «₽» в `texts.py`/`charges_list.py` убрать.
- **Выбор валюты (UX, TASK-0050):** шаг после суммы — кнопки 5 пресетов + «🌐 Другая валюта» →
  постраничный список всех валют (пагинация) и ввод кода/названия для фильтра (поиск по `code`/`name`).

## Последствия
- Положительные: оффлайн, версионируемый справочник; валюта у каждого списания; минимальная схема
  (2 колонки); переиспользуется форматтер.
- Отрицательные / компромиссы: нет конвертации между валютами (сознательно вне scope v1); ISO-список
  обновляется вручную; названия валют — на английском (RU-локализация названий — опционально позже).
- Разблокирует: TASK-0049 → TASK-0050. Миграции проверять на реальном Postgres (анти-дрейф).
