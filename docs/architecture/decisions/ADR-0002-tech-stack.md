# ADR-0002: Технологический стек бота

- **Статус:** Accepted
- **Дата:** 2026-05-31
- **Контекст сессии:** SESSION-2026-05-31-01
- **Связанные задачи:** TASK-0001

## Контекст
ТЗ §4 задаёт технические требования. Нужно зафиксировать стек как основу для
скелета проекта (TASK-0001), CI и `.gitignore`.

## Решение
Согласно ТЗ §4:
- **Язык:** Python 3 (целевая версия уточняется в TASK-0001, ориентир — 3.11+).
- **Фреймворк бота:** Aiogram 3.x (async).
- **БД:** SQLite на старте, с заложенным путём на PostgreSQL.
- **Планировщик:** APScheduler.
- **Качество:** `ruff` (формат+линт), `mypy` (типы), `pytest` (тесты) — в CI.
- **Конфигурация:** через окружение/`.env` (секреты вне репозитория, NFR-6).

## Последствия
- **+** Прямое соответствие ТЗ, зрелая экосистема, кроссплатформенность (NFR-7).
- **+** Async-стек хорошо ложится на бота + планировщик в одном процессе.
- **−** Миграция SQLite→PostgreSQL требует аккуратности с типами/JSON (отдельный ADR по слою данных).
- **Открыто:** конкретный слой доступа к данным (сырой `aiosqlite` vs. SQLAlchemy) —
  отдельным ADR в рамках задачи по БД.
