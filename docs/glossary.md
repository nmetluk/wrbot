# Глоссарий

| Термин | Значение |
|--------|----------|
| **Архитектор** | Роль в Cowork: ставит задачи, ревьюит, принимает решения, инициирует аудит. Код не пишет. |
| **Исполнитель** | Роль в Claude Code (машина с PAT): реализует задачи, пушит в GitHub. |
| **Источник правды** | `origin/main` на GitHub. Единственное авторитетное состояние проекта. |
| **Handoff** | Конвейер передачи задач через каталоги `handoff/` и git. |
| **Задача (TASK-XXXX)** | Markdown-файл с целью, критериями приёмки и контекстом; единица работы. |
| **Отчёт** | `handoff/reports/TASK-XXXX-report.md` — что и как сделано. |
| **Сессия** | Один рабочий заход одной роли; завершается логом в `sessions/`. |
| **Лог сессии** | `SESSION-YYYY-MM-DD-NN.md` — нарратив о ходе сессии. |
| **Состояние (state/)** | Машиночитаемый снимок проекта: `project.json`, `backlog.json`. |
| **ADR** | Architecture Decision Record — зафиксированное архитектурное решение. |
| **Майлстоун (Mx)** | Крупный раздел дорожной карты; после каждого — аудит. |
| **Аудит** | Независимая проверка (безопасность/архитектура/качество) в отдельной сессии. |
| **DoD** | Definition of Done — условия перевода задачи в `done/`. |
| **PAT** | Personal Access Token GitHub; есть на машине исполнителя для push. |
| **FSM** | Finite State Machine — пошаговые диалоги Aiogram. |
| **Списание / Charge** | Будущий платёж пользователя (основная сущность). |
| **tg_id** | Telegram user_id; ключ изоляции данных пользователя. |
