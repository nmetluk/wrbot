# wrbot v0.1.0 — Напоминания об оплатах

Первый релиз Telegram-бота, который ведёт список будущих списаний и присылает
напоминания об оплате в выбранное вами время. Полностью покрыт объём v1 технического
задания (§1–§5).

## ✨ Возможности

- **Списания.** Пошаговое создание (название → сумма → кошелёк → категория → дата →
  периодичность → уведомления), список ближайших платежей, редактирование, удаление,
  кнопка «Отметить оплаченным» (для регулярных — дата сдвигается на следующий период,
  одноразовые закрываются).
- **Периодичность:** одноразово, ежемесячно, ежеквартально, ежегодно (с корректным
  переносом конца месяца, напр. 31-е → последний день февраля).
- **Справочники:** кошельки/карты и категории — полный CRUD в «Настройках».
- **Умные напоминания:** приходят в личный чат в назначенное время, с кнопками
  «Оплачено», «Отложить», «Редактировать». «Отложить» переносит напоминание, не трогая
  дату платежа.
- **Гибкая настройка уведомлений:** время напоминаний и набор дней (за сколько дней
  напоминать) — глобально и индивидуально для каждого списания.
- **Часовой пояс:** выбор пояса в настройках — напоминания приходят в корректное
  локальное время.
- **Изоляция данных:** у каждого пользователя свои кошельки, категории и списания.
- Интерфейс полностью на русском. Команды: `/start`, `/help`, `/cancel`.

## 🛠 Технологии

Python 3.11+ · aiogram 3.x (long polling) · SQLAlchemy 2.0 (async) + Alembic ·
APScheduler · SQLite (старт) / PostgreSQL.

## 🔒 Надёжность

- Работа 24/7: устойчивость к перезапуску, периодический свип напоминаний по БД —
  **идемпотентный** (без задвоений после рестарта), точность отправки ±1 минута.
- Глобальный обработчик ошибок и graceful shutdown (SIGINT/SIGTERM).
- Секреты — только в окружении (`.env`), не в репозитории и не в Docker-образе.

## 🚀 Запуск

```bash
cp .env.example .env          # заполните BOT_TOKEN (от @BotFather)
docker compose up -d --build  # SQLite по умолчанию; миграции применяются автоматически
```

Вариант с PostgreSQL и подробности — в [`docs/deployment.md`](../deployment.md).

## ✅ Качество

- Полный CI на зафиксированных версиях инструментов (`uv.lock`): `ruff`, `mypy --strict`,
  `pytest`, миграции Alembic, валидация состояния.
- 179 тестов, включая интеграционные (жизненный цикл сессии, маршрутизация) и сквозной
  e2e через `Dispatcher` с проверкой сохранения данных.

## ⚠️ Примечания

- Перед боевым запуском рекомендуется ручной смоук-прогон в реальном Telegram
  (чеклист — в отчётах QA).
- Healthcheck в `docker-compose` базовый (проверка живости процесса).
