# wrbot v0.3.0 — Доработки по отзывам пользователей

Третий релиз. Майлстоун **M7**: исправлено отображение списаний и добавлены пять улучшений,
о которых просили реальные пользователи — от иконок кошельков до дублирования напоминаний в каналы.

## 🐞 Исправлено

- **Корректное отображение списаний.** В подтверждении создания и в карточке теперь видны реальные
  **название кошелька и категории** (раньше были заглушки/ID), дата в формате **ДД.ММ.ГГГГ**,
  период и тип уведомлений — по-русски. Тот же аккуратный формат — в списке и в текстах напоминаний.

## ✨ Что нового

- **Проверка даты по периоду.** Сначала выбираете периодичность, затем дату — и бот не даёт указать
  дату вне разумного окна (для ежемесячного — не дальше месяца, для ежегодного — не дальше года),
  подсказывая допустимый диапазон.
- **«Мои списания» с группировкой по категориям.** Краткий обзор: по категориям, с ближайшей датой
  платежа и кнопками перехода. Списания без категории — отдельным блоком.
- **Иконки кошельков/карт.** Выбор эмодзи-иконки (👛 💰 🏦 💳 💵 🪙) при добавлении и редактировании —
  чтобы визуально различать карты и кошельки в списках и карточках.
- **Дублирование напоминаний в каналы/группы.** В настройках категории можно указать chat_id каналов
  или групп, куда будут дублироваться напоминания о списаниях этой категории. Если у бота нет прав
  в чате — придёт уведомление, а остальные чаты получат напоминание как обычно.
- **Удобное редактирование.** При изменении списания над диалогом «висит» живая карточка текущих
  данных и обновляется на каждом шаге — ничего не теряется и не засоряет переписку.

## 🛠 Технологии

Python 3.11+ · aiogram 3.x · SQLAlchemy 2.0 (async) + Alembic · APScheduler ·
SQLite (старт) / PostgreSQL.

## 🚀 Обновление

```bash
git pull
docker compose up -d --build   # применит миграции (wallets.icon, categories.notify_chat_ids) автоматически
```

Миграции добавляют две колонки безопасно (с дефолтом/backfill). **Перед обновлением на проде —
свежий бекап БД** (см. `docs/deployment.md`).

## ✅ Качество

- 240 тестов; полный CI на зафиксированных версиях (`uv.lock`): `ruff`, `mypy --strict`,
  `pytest`, миграции Alembic (проверены `alembic check` и round-trip), валидация состояния.
- Принят аудитом майлстоуна (`handoff/reports/AUDIT-M7-2026-06-04.md`).

## ⚠️ Примечания

- Дублирование в каналы/группы: добавьте бота в чат и дайте право публикации; в v1 поддерживается
  числовой chat_id.
- Рекомендуется живой смоук в реальном Telegram (особенно дубли в канал/группу).
