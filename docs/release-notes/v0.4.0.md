# wrbot v0.4.0 — ID группы и фикс настройки каналов

Доработки вокруг дублирования напоминаний в каналы/группы: удобное получение ID чата и
исправление зависавшей кнопки добавления канала.

## ✨ Что нового

- **Получение ID группы — `/getgrid` и @упоминание.** Добавьте бота в нужную группу или канал-
  обсуждение и вызовите там **`/getgrid`** (или просто **@упомяните бота**) — бот пришлёт числовой
  ID чата, готовый к копированию. Этот ID вставляется в «⚙️ Настройки → 🏷️ Категории →
  🔔 Каналы для напоминаний» (кнопка «➕ Добавить канал/группу»). Вызвать может любой участник.
  Команда `/getgrid` видна в меню команд групп; в личном чате бот подскажет, что команда — для групп.

## 🐞 Исправлено

- **Кнопка «➕ Добавить канал/группу» больше не зависает.** В настройках уведомлений категории
  добавление и удаление целей теперь срабатывает корректно — устранён конфликт обработчиков,
  из-за которого нажатие не давало реакции.

## 🛠 Технологии

Python 3.11+ · aiogram 3.x · SQLAlchemy 2.0 (async) + Alembic · APScheduler ·
SQLite (старт) / PostgreSQL.

## 🚀 Обновление

```bash
git pull
docker compose up -d --build
```

Миграций БД в этом релизе нет. Privacy mode бота отключать не нужно — Telegram и так доставляет
боту команды и @упоминания в группах.

## ✅ Качество

- 241 тест; полный CI на зафиксированных версиях (`uv.lock`): `ruff`, `mypy --strict`, `pytest`,
  миграции Alembic, валидация состояния.
- Добавлены router-level регресс-тесты (через `dp.feed_update`) на маршрутизацию настроек категории
  и на `/getgrid`/@упоминание — чтобы класс «тихих» баг-роутингов больше не проходил незамеченным.

## ⚠️ Примечания

- `/getgrid` работает в группах/супергруппах; в v1 для настроек поддерживается числовой chat_id.
- Рекомендуется живой смоук: `/getgrid` в группе и добавление полученного ID в настройки категории.
