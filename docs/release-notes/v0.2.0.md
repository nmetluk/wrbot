# wrbot v0.2.0 — Наблюдаемость и админ-функции

Второй релиз. Майлстоун **M6**: бот теперь сам присматривает за собой — делает бэкапы,
ведёт аудит-лог и шлёт администратору статус и ежедневную статистику в приватный канал.
Для обычных пользователей интерфейс не изменился (кроме мелкой UX-полировки) — всё новое
работает «за кулисами».

## ✨ Что нового

- **Приватный админ-канал.** Бот публикует в выделенный Telegram-канал статусы операций,
  ошибки и сводки. Включается через `ADMIN_CHANNEL_ID`/`ADMIN_TZ`; без настройки канала
  всё работает как раньше (тихий no-op). Секреты (`BOT_TOKEN`, строка БД) в канал не попадают.
- **Автоматические бэкапы.** Ежечасно создаётся резервная копия базы (SQLite `VACUUM INTO`
  или `pg_dump`), хранится 36 последних копий с авто-ротацией. В канал приходит подтверждение
  («✅ Бэкап ОК» / «❌ Ошибка»), heartbeat и краткая сводка.
- **Аудит-лог.** Действия пользователей и системы фиксируются в отдельной таблице (только
  тип события и идентификатор — без сумм, имён и текстов). Критичные ошибки дублируются в канал.
- **Ежедневный дашборд в 21:00** (в часовом поясе администратора). Рост пользователей,
  активность, платежи и периодичности, топ-категории, напоминания и ошибки — текстовая сводка
  плюс графики (matplotlib).

## 🔧 Улучшения

- Убрана дублирующая кнопка «Закрыть» в списке «Мои списания»; «Отмена» теперь работает только
  в активных диалогах и всегда возвращает в главное меню с пометкой «Действие отменено».

## 🛠 Технологии

Python 3.11+ · aiogram 3.x · SQLAlchemy 2.0 (async) + Alembic · APScheduler ·
SQLite (старт) / PostgreSQL · matplotlib (графики дашборда).

## 🔒 Надёжность и приватность

- Бэкапы и тяжёлый рендеринг графиков выполняются в отдельных потоках — событийный цикл бота
  не блокируется.
- Наблюдаемость безопасна по умолчанию: без канала — no-op, ошибки отправки изолированы,
  секреты санитизируются, аудит-лог не хранит чувствительных данных.

## 🚀 Обновление

```bash
git pull
docker compose up -d --build   # применит миграцию audit_log автоматически
```

Новые переменные (опционально, для админ-канала) — в `.env.example`:
`ADMIN_CHANNEL_ID`, `ADMIN_TZ`. Подробности — в [`docs/deployment.md`](../deployment.md).

## ✅ Качество

- 208 тестов; полный CI на зафиксированных версиях (`uv.lock`): `ruff`, `mypy --strict`,
  `pytest`, миграции Alembic, валидация состояния.
- Тест-сьют детерминирован (исправлена зависимость от порядка тестов в проверке бэкапа).
- Принят повторным аудитом майлстоуна (`handoff/reports/AUDIT-M6-2026-06-03.md`).

## ⚠️ Примечания

- Админ-функции опциональны: без `ADMIN_CHANNEL_ID` бот работает как в v0.1.x.
- Перед боевым запуском рекомендуется ручной смоук-прогон в реальном Telegram.
