# Требования (структурировано, с трассировкой к ТЗ)

Каждое требование имеет ID для ссылок из задач, отчётов и тестов.
Колонка «ТЗ» указывает раздел исходного `spec/spec-v1.md`.

## Функциональные требования (FR)

| ID | Требование | ТЗ |
|----|-----------|----|
| FR-1 | Команда `/start` показывает главное меню с кнопками: ➕ Новое списание, 📋 Мои списания, ⚙️ Настройки, ❔ Помощь | §3.1 |
| FR-2 | Команды `/cancel` (прервать диалог) и `/help` (справка) | §4 |
| FR-3 | Пошаговое создание списания: название → сумма → кошелёк (с «Добавить новый») → категория (или пропустить) → дата → периодичность → настройка уведомлений | §3.2 |
| FR-4 | Периодичность: одноразово / ежемесячно / ежеквартально / ежегодно | §2, §3.2 |
| FR-5 | «Мои списания»: список ближайших платежей, отсортированный по дате; по каждому — название, сумма, кошелёк, категория, следующая дата | §3.2 |
| FR-6 | Действия над списанием: Редактировать, Удалить, Отметить как оплачено | §3.2 |
| FR-7 | «Отметить как оплачено» сдвигает `next_date` на следующий период (кроме одноразовых, которые закрываются) | §3.2 |
| FR-8 | Настройки → Кошельки/карты: полный CRUD | §3.3 |
| FR-9 | Настройки → Категории: полный CRUD | §3.3 |
| FR-10 | Глобальные уведомления: время (по умолчанию 10:00) и дни напоминаний (комбинации из 5/3/1) | §3.3 |
| FR-11 | Списание может иметь индивидуальные дни напоминаний (`individual_days`) либо наследовать глобальные, либо быть отключённым | §2, §3.2, §3.4 |
| FR-12 | В назначенное время бот шлёт напоминание в личный чат: название, сумма, кошелёк, дата + кнопки «Оплачено» / «Отложить» / «Редактировать» | §3.4 |
| FR-13 | Все данные хранятся изолированно по `tg_id` пользователя | §1.3 |
| FR-14 | Интерфейс и тексты — только на русском (первый этап) | §4 |

## Решённые вопросы (закрыты ADR в SESSION-2026-06-01-01)
- **Валюта/формат суммы** → `Decimal` (2 знака), одна валюта ₽ в v1 — **ADR-0006**.
- **«Отложить»** → не сдвигает `next_date`; повтор напоминания на следующий день — **ADR-0005**.
- **Часовой пояс** → поле `tz` у пользователя, дефолт `Europe/Moscow`; `notify_time` локально в `tz` — **ADR-0004**.
- **Слой данных и миграции SQLite→PostgreSQL** → SQLAlchemy 2.0 async + Alembic — **ADR-0003**.
- **Жизненный цикл списания** (мягкое закрытие одноразовых, кламп конца месяца) — **ADR-0006**.
- **Движок уведомлений** (свип по БД, идемпотентность через `sent_reminders`, рестарт-безопасность) — **ADR-0005**.
- **Доставка апдейтов** → long polling в v1 — **ADR-0007**.
