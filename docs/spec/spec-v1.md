# Техническое задание (ТЗ): Telegram-бот «Напоминания об оплатах»

> **Источник требований.** Этот документ — зафиксированное исходное ТЗ. Он
> **не редактируется**. Любые уточнения и изменения оформляются как новая версия
> (`spec-v2.md`) и/или через ADR в `docs/architecture/decisions/`.

**Версия:** 1.0
**Дата:** 31 мая 2026
**Назначение:** Простой бот-напоминалка о предстоящих платежах и списаниях.

## 1. Общие положения
**1.1 Назначение бота**
Бот позволяет пользователю вести список будущих списаний, группировать их по
категориям и кошелькам/картам, а также получать timely напоминания за несколько
дней до даты платежа.

**1.2 Платформа**
Telegram-бот (Telegram Bot API).

**1.3 Основные принципы**
- Максимальная простота интерфейса (inline-кнопки, пошаговые диалоги).
- Всё хранится в базе данных отдельно для каждого пользователя (по Telegram `user_id`).
- Уведомления приходят в личный чат с ботом.
- Бот работает 24/7, уведомления отправляются в указанное пользователем время.

## 2. Основные сущности
| Сущность | Поля | Примечание |
|----------|------|-----------|
| **Пользователь** | `tg_id`, `notify_time` (по умолчанию 10:00), `global_days` (массив: 5, 3, 1) | Глобальные настройки |
| **Кошелёк / Карта** | `id`, `user_id`, `name` (например «Тинькофф Black», «Сбер», «ЮMoney») | Создаётся в настройках |
| **Категория** | `id`, `user_id`, `name` (например «Подписки», «ЖКХ», «Кредиты») | Создаётся в настройках |
| **Списание** | `id`, `user_id`, `name`, `amount`, `wallet_id`, `category_id` (опционально), `next_date`, `period` (одноразово / ежемесячно / ежеквартально / ежегодно), `individual_days` (или `null`) | Основная сущность |

## 3. Функциональные требования
### 3.1 Главное меню (`/start`)
Кнопки:
- ➕ **Новое списание**
- 📋 **Мои списания**
- ⚙️ **Настройки**
- ❔ **Помощь**

### 3.2 Работа со списаниями (CRUD)
**Создание списания** (пошагово):
1. Название
2. Сумма
3. Кошелёк/карта (выбор + кнопка «Добавить новый»)
4. Категория (выбор или пропустить)
5. Дата следующего списания
6. Периодичность
7. Уведомления (глобальные / свои / отключить)

**Мои списания**:
- Список ближайших платежей (сортировка по дате).
- Для каждого: название, сумма, кошелёк, категория, следующая дата.
- Действия: Редактировать / Удалить / **Отметить как оплачено**.

**Отметить как оплачено** → дата автоматически сдвигается на следующий период
(если не одноразовое).

### 3.3 Настройки
Разделы:
- **Кошельки / карты** — полный CRUD
- **Категории** — полный CRUD
- **Глобальные уведомления**
  - Время уведомлений (по умолчанию 10:00)
  - Дни напоминаний (за 5, 3, 1 день — любые комбинации)

### 3.4 Система уведомлений
- Уведомления приходят в выбранное время.
- Поддерживаются глобальные и индивидуальные настройки дней напоминаний.
- В уведомлении: название, сумма, кошелёк, дата + кнопки «Оплачено» / «Отложить» / «Редактировать».

## 4. Технические требования
- **Язык:** Python 3
- **Фреймворк:** Aiogram 3.x (рекомендуется)
- **БД:** SQLite (для старта) / PostgreSQL
- **Планировщик:** APScheduler
- Команды: `/start`, `/cancel`, `/help`
- Только русский язык на первом этапе

## 5. Нефункциональные требования
- Стабильная работа 24/7
- Уведомления с точностью ±1 минута
- Чистый, хорошо документированный код
- Возможность лёгкого расширения

## 6. Доработки второй итерации (не входит в это ТЗ)
- Экспорт в CSV/PDF
- Статистика по категориям и кошелькам
- Несколько валют
- Семейный/групповой режим
